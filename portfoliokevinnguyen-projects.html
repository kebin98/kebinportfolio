Thunderbolt Inferno - Fighting Game

I've dreamed of making fighting games ever since I was a young child - and started my first attempts at the endeavour when I was 11 years old.


Constantly I would be curious in programming, drawing and animating my characters with unique setpieces and movesets.


Over the years I focused less on a high quantity of characters and more on mechanical depth, to make sure the gameplay felt fluid and responsive.


This fighting game was my first proper Game Maker Studio 2 project - as a way for me to understand coding in the language.


It was developed betwen April 13th and July 21st before I decided to stop working on the project.


The most significant aspect of the coding that helped me program characters faster was building a state system that allowed the backbone of most character techniques to share the same code.


Animating the characters was an incredibly extensive endavour - one that would mean continuing the game would only be feasible if more animators were on board with this type of project.

Thunderbolt Engine

I had a very huge scope for a massive adventure game during November.


Due to this I started to experiment with a map - generation system where you could develop platforming maps using an editor which can be stored in a text file.


This is how the development for the Thunderbolt Engine began.


With this project I hope to make an intuitive game editor that will allow for editing game maps, with a unique enemy editor that will allow for making your own unique enemy AI.


Currently the engine itself is in its infancy and a lot of my resources are going towards making a video game project with various different gameplay styles.


I created a lot of algorithms to allow for the Thunderbolt Engine to work the way it does, and I plan to program more in the future.


Engine Features

- The Thunderbolt Engine currently uses various grid systems to store different properties for the maps that are created.


- A generator mode was created so that aspects of the map's terrain such as the floor depth and slopes - are generated automatically based on its relative positioning compared to other blocks of its kind on the grid.


- Loading and saving was made with its own UI - maps are stored within various text files, allowing for relatively fast grid loading.


- An RGB colour picker was made by hard-coding some number conversion, to allow denary values to be converted to binary and hexadecimal.


- A layer system was created to allow for parallax scrolling between cosmetic maps.


- Filling in gaps on the map was made possible - special thanks to Smogzone for the inspiration of this tool..


- To reduce lag on loading huge maps, a window generating system was implemented - where parts of the map would generate based on its relative positioning around a central point, a "window" per se.


No current demo exists for the Thunderbolt Engine however it is still being worked on gradually.
